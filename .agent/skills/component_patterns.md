# Component Patterns

> **Purpose:** Quick-reference catalog of components created or significantly modified in this project. Append a new entry whenever a component is built or substantially changed. Enables reuse and consistency across sessions.
>
> **Format per entry:**
> - **Name & location** — file path
> - **Purpose** — one-line description
> - **Props / API** — key props, types, defaults
> - **Internal structure** — state, effects, rendering approach
> - **Styling approach** — tokens, classes, or inline styles used and why
> - **Reuse notes** — when and how to reuse this pattern
